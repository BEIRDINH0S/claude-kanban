Why claude-kanban

The thing other Claude Code wrappers can't copy.

Won't get you banned

No OAuth impersonation. Period.

Most Claude Code wrappers reuse the official client_id to call api.anthropic.com directly — that's TOS-violating impersonation, detectable by Anthropic, and grounds for suspending your Max/Pro subscription. claude-kanban drives the official claude binary in a hidden PTY, so every request looks identical to a normal CLI user.

Parallel by default

One card, one Claude session. Run a dozen.

The Node sidecar multiplexes query() calls from the official Anthropic Agent SDK. Kick off three refactors and walk away — the kanban tracks each session's state and moves cards automatically.

No terminal babysitting

Approve tools with a click.

Every canUseTool request surfaces in the UI: see the command, click approve / refuse / always-allow. No more chained CLI prompts to validate one at a time.

Lifecycle

One card, five states. Zero hand-holding.

You create the card, you walk away. The app listens to SDK events and moves the card to the right column without you touching it.

01 · Todo

You create a card

Title + working directory. The session is created and ready to receive its first prompt.

02 · In progress

Claude is working

First message sent → card in in_progress. You see tool-uses and a live activity indicator on the card.

03 · Review

Permission requested

The SDK fires canUseTool. The card jumps to Review with an Approve / Refuse popup. You click, Claude resumes.

04 · Idle

Turn ended

SDK result event. Claude is waiting for a follow-up. The card is paused; you resume whenever.

05 · Done

You archive

Drag manually to Done once the PR is merged or the result is validated. The session is stopped cleanly.
