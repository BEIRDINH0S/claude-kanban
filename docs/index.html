Ce qu'on a vraiment construit

Onze choses que Claude Code ne fait pas tout seul.

Les features qu'on a passé du temps à coder. Le terminal, le tool-use, la complétion — c'est Claude Code. Le reste, c'est l'app.

01 — Parallèle

Autant de sessions Claude que tu veux. En parallèle.

Une carte = une session. Le sidecar Node multiplexe les query() du SDK Anthropic. Tu lances trois refactors et tu pars chercher un café — ils continuent en parallèle. Aucun cap fixé par l'app.

03 — Worktrees

Un git worktree par carte. Zéro conflit.

Quand tu crées la carte, l'app crée un worktree dédié dans .claude-kanban-worktrees/ sur une branche claude-kanban/card-…. Tes Claude éditent en parallèle, jamais le même fichier deux fois. Badges live ahead / behind / dirty par carte.

Cycle de vie

Une carte, cinq états. Aucune main d'œuvre.

Tu crées la carte, tu pars. L'app détecte les events du SDK et déplace la carte dans la bonne colonne sans que tu touches à rien.

01 · Todo

Tu crées une carte

Titre + dossier de travail. La session est créée, prête à recevoir le premier prompt.

02 · En cours

Claude bosse

Premier message envoyé → carte en in_progress. Tu vois les tool-uses et la preview live.

03 · Review

Permission demandée

Le SDK déclenche canUseTool. La carte saute en Review, popup avec Approuver / Refuser. Tu cliques, ça redémarre.

04 · Idle

Tour fini

Event result du SDK. Claude attend un follow-up. La carte est en pause, tu reprends quand tu veux.

05 · Done

Tu archives

Drag manuel vers Done quand t'as merge la PR ou validé le résultat. La session est arrêtée proprement.
